# Step 1. Complexity Diagnosis

## 관찰 요약
- **과도한 동시 노출 요소**: 상단+좌측 카드에 고급 제어(LLM/API/OAuth/실행/시트/전이검증)가 한 번에 노출되어 초기 진입 인지부하가 큼.
- **중복 정보**: 진행 상태(자동생성/실행/ops stage/oneclick progress)가 여러 영역에서 반복 표현.
- **1화면 내 결정 포인트 수**: 주요 CTA 및 토글/체크/필터 포함 체감 결정 포인트가 20+로 높음.
- **스크롤 피로 요소**: 단일 화면에 진단/실행/고급검증/로그까지 수직 누적.
- **버튼/CTA 혼선**: 실행 계열 버튼(분석/자동생성/실행/확정)이 유사 비주얼로 병렬 배치되어 순서 인지가 어려움.
- **시각적 우선순위 불명확**: H 계층이 약하고 카드 간 위계보다 기능 나열 중심.

## cognitive_load_score
- **Before: 8.4 / 10**

## 개선 방향
1. 상단을 ‘URL + 원클릭’ 중심으로 재구성
2. 고급 제어는 기본 접힘(Progressive Disclosure)
3. 버튼 역할별(Primary/Secondary) 명확화
4. 화면 목적별 구역 구분(실행, 결과, 상세)
