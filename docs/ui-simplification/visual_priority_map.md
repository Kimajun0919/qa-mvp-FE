# Step 3. Visual Hierarchy Optimization

## H1 > H2 > H3
- H1을 명시(`QA Agent Console`)
- 카드 제목(H2) 중심 구조 유지
- 상태/보조 블록은 H3/본문 수준으로 후퇴

## Primary CTA 정책
- 동일 뷰포트 내 **원클릭 전체 QA 실행**만 Primary 컬러 유지
- 자동생성 버튼 등은 Secondary로 조정

## Role-based colors
- Primary: brand(실행)
- Neutral: secondary(조회/보조)
- Danger: 기존 fail/red 계열 유지(에러/경고)

## 버튼 3개 초과 영역
- 상단/고급 제어 버튼은 그룹화(`btn-group`) 및 disclosure 내부 배치

## 카드 분리 및 노이즈 정리
- Hero card 추가로 진입 문맥 분리
- 고급 제어를 접어서 경계선/강조 경쟁 감소
