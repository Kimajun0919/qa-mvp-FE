# Step 5. Interaction Simplification

## Multi-input → Stepper
- 상단에 3단계 가이드(입력→실행→결과) 명시
- 기존 기능 호출 순서는 유지

## Inline validation
- 필수 입력 경고 문구를 행동 유도형으로 변경
  - URL 입력 누락: 예시 URL 포함 안내
  - 화면명 누락: 재시도 문구 + 입력 예시

## Failure message 개선
- 단순 에러 문구 대신 다음 행동 제시(예: URL 입력 후 재실행)

## One-screen-one-purpose
- 상단: 실행 시작
- 중단: 결과/의사결정
- 하단: 맵/로그 탐색
